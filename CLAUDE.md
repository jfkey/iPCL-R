## Project Overview

iPCL-R (Intelligent Pre-trained Chip Layout Routing) is a machine learning framework for automated chip routing pattern generation. It treats routing patterns as sequences that can be learned and generated by large language models, using a 3-stage pipeline: tokenization → training → evaluation.

## Common Commands

### Pipeline Execution

```bash
# Generate pipeline configuration
python -m flow.pipeline_init --create-flow-config config.json

# Stage 1: Tokenization (single process)
python -m flow.launch_tokenization --flow-config config.json

# Stage 2: Training (distributed)
accelerate launch -m flow.launch_training --flow-config config.json

# Stage 3: Evaluation (distributed)
accelerate launch -m flow.launch_evaluation --flow-config config.json
```

### Data Synthesis (Optional)

```bash
python -m data_synthesis.main_aggregation --design_list nvdla shanghai_MS --output_dir /path/to/output
```

### Experiments

```bash
# Tokenizer comparison
python -m experiments.tokenizer_comparison.init_env --work-dir /path/to/work_dir

# LLM fine-tuning
accelerate launch -m experiments.sft_llm.training
python -m experiments.sft_llm.evaluation

# GRPO fine-tuning
accelerate launch -m experiments.grpo_ft.training
```

## Key Classes

### Tokenization (Stage 1)
- **UnifiedTokenizer** (`flow/tokenization/tokenizer.py`): Supports 5 algorithms (DecimalWordLevel, Seg-BPE, Concat-BPE, Seg-BBPE, Concat-BBPE). Converts 3D coordinates to directional tokens (R/L/U/D/T/B).
- **TokenizationPipeline** (`flow/tokenization/pipeline.py`): Orchestrates tokenization workflow.

### Training (Stage 2)
- **TrainingPipeline** (`flow/training/pipeline.py`): Uses T5-Gemma encoder-decoder with HuggingFace Accelerate for distributed training.

### Evaluation (Stage 3)
- **EvaluationPipeline** (`flow/evaluation/pipeline.py`): Multi-metric assessment (ROUGE, BLEU, RED), tree structure validation, DEF format output.

### Configuration
- **FlowConfig** (`flow/config.py`): Hierarchical dataclass-based configuration with DatasetConfig, TokenizationStageConfig, TrainingStageConfig, EvaluationStageConfig.

### Utilities
- **CoordinatePoint** (`flow/utils/token_preprocessing.py`): Immutable 3D point (x, y, metal_layer) with arithmetic operations.
- **SpecialTokenManager** (`flow/utils/special_tokens.py`): Manages CORE/SOURCE/TREE/OVERLAP tokens.
- **Constants** (`flow/utils/constants.py`): DIRECTION_MAP (R/L/U/D/T/B), coordinate patterns.

## Domain Concepts

### Direction Encoding
Routing movements are encoded as directional tokens:
- `R/L`: Right/Left (±X axis)
- `U/D`: Up/Down (±Y axis)
- `T/B`: Top/Bottom (±metal layer)

### Special Tokens
- **CORE**: `<BOS>`, `<EOS>`, `<PAD>`, `<SRC_END>`
- **SOURCE**: `<DRIVER>`, `<LOAD>` (routing semantics)
- **TREE**: `<PUSH>`, `<POP>`, `[BRANCH]`, `[END]` (tree structure)

### Data Flow
```
EDA Files → Data Synthesis → HuggingFace Dataset
    ↓
Stage 1: Corpus → UnifiedTokenizer → Token Dataset
    ↓
Stage 2: Token Dataset → T5Gemma Model → Checkpoints
    ↓
Stage 3: Model → Inference → Metrics + DEF Output
```

## Configuration System

All stages are configured via a single JSON config file (`config.json`). Use `python -m flow.pipeline_init --create-flow-config config.json` to generate a template.

Key configuration sections:
- `dataset`: Source (hub/local), splits, paths
- `tokenization`: Algorithm, vocab size, max sequence length
- `training`: Model architecture, hyperparameters, distributed settings
- `evaluation`: Generation config, metrics, output format

## Distributed Training

Uses HuggingFace Accelerate for multi-GPU training:
```bash
# Configure accelerate (first time)
accelerate config

# Launch distributed training/evaluation
accelerate launch -m flow.launch_training --flow-config config.json
accelerate launch -m flow.launch_evaluation --flow-config config.json
```

## Dataset

- **HuggingFace Hub**: `AiEDA/iPCL-R` (default)
- **Data types**: `net_seqs`, `pin2pin_pattern_seqs`, `pin2pin_loc_seqs`, `design_graph`
- **Format**: Parquet files consolidated into HuggingFace Dataset
